Zope Page Templates for .NET

Zope Page Templates (ZPT) is a neat attribute-based template language for creating XHTML and HTML5 documents from dynamic data. From an ASP.NET MVC perspective, ZPT-Sharp is a customised ViewEngine that would be used instead of the default 'aspx' ViewEngine.

What makes ZPT so great?

The best way to describe why ZPT is such a zen templating language is to suggest you take a look at the ZPT example and see for yourself. Some of the more exciting advantages it offers over the classic ASP.NET templating engine though are:

  • Raw ZPT source files are themselves valid XHTML (or HTML5) documents; all of the logic of the ZPT language is handled by well-formed XML attributes. This means that the source files can be rendered by web browsers and editing tools 'offline' from the web application.
  • There is no code and no obscure/ugly placeholder content visible when the source files are viewed 'offline' in a web browser or editing tool (unlike aspx's <% %> and <# #>). If you wish to provide placeholder content for dynamic data, this placeholder content can be anything you choose (the placeholder content is replaced when the page is rendered by the ZPT engine).
  • ZPT is very MVC-oriented and does not try to provide functionality beyond that which is needed for the role of a view. It encourages best practice and does not allow you to mix business logic directly into your markup.
  • The syntax is easy to learn and should be intuitively legible to almost everybody.

So what's a Zope anyway?

Zope is an open source web application framework written in Python. A crude parallel would be to think of Zope as equivalent to the .NET framework plus IIS together, along with a built-in object database and a web-management front-end. The ZPT language was originally developed as a feature of Zope but it does not depend on Zope to function.
