Introduction to TAL

TAL is how ZPT performs rendering logic upon a page. Where METAL brings together the components of that page through reusable pieces of markup defined in macros, TAL is used to bind the model to the page. It can inject data from the application, repeat sections of markup, control attributes, conditionally render sections of markup and more.

TAL attributes

The following attributes exist in the TAL namespace, activated using the tal: attribute prefix.

Attribute Summary
define Define a TALES variable
condition Do not render the element (or any of its children) if a condition evaluates to false
repeat Repeat the element (and all of its children) for each item within a collection
content Replace all child nodes of the current element with the results of a TALES expression (mutually exclusive with replace)
replace Replace the current element, along with all of its child nodes with the results of a TALES expression (mutually exclusive with content)
attributes Add, remove or set the value of HTML/XML attributes on the current element
omit-tag Do not render the element (but preserve the element's children) if a condition evaluates to true
on-error Conditionally render some markup if an exception is caught whilst rendering the current element or its children.

Order of processing

TAL attributes are processed in the order in which they are listed in the above table. If an attribute indicates that it ceases TAL processing (for example, a tal:condition attribute means that an element is not to be rendered), then subsequent TAL attributes will not be processed.

The exception to this is tal:on-error attributes. On-error attributes are processed immediately when an error is raised; that error will then interrupt the processing of other TAL attributes. Thus, on-error attributes lie outside the normal order of attribute processing.
