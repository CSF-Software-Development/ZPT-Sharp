bool CraigFowler.Web.ZPT.Tales.TalesExpression.GetBooleanValue ( )

Invokes GetValue and then returns the boolean equivalent of the resolved value.

From the TALES specification 1.3 - http://wiki.zope.org/ZPT/TALESSpecification13 relating to 'not' expressions:

If the expression supplied does not evaluate to a boolean value, not will issue a warning and coerce the expression's value into a boolean type based on the following rules:

  1. integer 0 is false
  2. integer > 0 is true
  3. an empty string or other sequence is false
  4. a non-empty string or other sequence is true
  5. a non-value (e.g. void, None, Nil, NULL, etc) is false
  6. all other values are implementation-dependent

If no expression string is supplied, an error should be generated.

This implementation further defines that:

  1. Any nonzero number (or that does not become zero upon casting to a System.Single) is true. This includes negative numbers.
  2. Item 5 in the list above (checking for null values) is performed before items 3 and 4.
  3. Item 4 refers to sequences. In this implementation then an object is considered a sequence if it implements ICollection. In this case its ICollection.Count property is checked to determine its length.
  4. An implementation-specific step that is taken is the attempt to use the IConvertible interface also with Convert.ToBoolean(Object) to perform the conversion to boolean.
  5. Finally, if the value still cannot be coerced to Boolean by any other means, false is returned.
Returns:
A System.Boolean
