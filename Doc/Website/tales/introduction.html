Introduction to TALES

TALES describes and evaluates expressions which describe the data model passed to your ZPT views. In MVC syntax, these expressions access your model.

Expressions are strings, which are included within the TAL or METAL attribute values at applicable points.

Expression prefixes

Each expression optionally begins with a prefix - a short identifier followed by a colon - indicating how it should be interpreted. Expressions which omit the prefix are treated as path expressions, which is the default expression-type. The available prefixes are as follows:

Prefix Description
path: Evaluates a value from the current TALES context.
string: Builds a string value.
not: Evaluates a value from an expression following the prefix, gets the boolean equivalent of that value and negates it.

Boolean coercion

Some TAL attributes interpret the value of an expression as a boolean. TALES defines a set of rules for coercing non-boolean values into a boolean equivalent. Note that these rules differ from the manner in which a not: expression interprets values as boolean.
