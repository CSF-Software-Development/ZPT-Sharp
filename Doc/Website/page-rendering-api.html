ZPT-Sharp rendering API

ZPT-Sharp provides an API to render XML/HTML pages using ZPT.

Assemblies

There are two assemblies relevant to rendering pages using ZPT-Sharp.

The CSF.Zpt.Api assembly contains the interfaces and public-visibility types which are used by the API. It has no dependencies of its own except upon the .NET framework and is safe to reference from your project.

The CSF.Zpt assembly contains the implementation for rendering pages and (if you are using dependency-injection), might only be referenced by your DI wirings assembly (if you use this pattern).

Abstract types & interfaces

The following types are all found in the CSF.Zpt.Api assembly.

IZptDocumentFactory

IZptDocumentFactory provides the public interface for creating instances of IZptDocument. 

using System.IO;
using CSF.Zpt;

FileInfo sourceFile = GetSourceFile();
IZptDocumentFactory factory = GetFactory();
IZptDocument myDoc = factory.CreateDocument(sourceFile);

IZptDocument

IZptDocument provides the public interface for rendering documents. 

using System.IO;
using CSF.Zpt;

IZptDocument myDoc = GetDocument();
string outputPath = GetOutputPath();

using(var outputStream = File.Open(outputPath))
using(var writer = new StreamWriter(outputStream))
{
    myDoc.Render(writer);
}

RenderingOptions

RenderingOptions wraps the available options for rendering a IZptDocument. If you wish to use non-default options then you may provide them as an optional parameter to IZptDocument.Render. 

using System.IO;
using CSF.Zpt;
using CSF.Zpt.Rendering;

IZptDocument myDoc = GetDocument();
string outputPath = GetOutputPath();
RenderingOptions options = GetOptions();

using(var outputStream = File.Open(outputPath))
using(var writer = new StreamWriter(outputStream))
{
    myDoc.Render(writer, options);
}

Implementation types

The following types are all found in the CSF.Zpt assembly.

ZptDocumentFactory

ZptDocumentFactory is the concrete implementation of IZptDocumentFactory. It works upon either files in a filesystem or Stream instances. In the case of files, it can typically detect the file type (XML or HTML) from its extension. It then returns an implementation of IZptDocument suitable to that file.

The constructor to this type also permits forcing the factory into creating documents only of a specific type.

ZptHtmlDocument & ZptXmlDocument

Generally, it is ill-advised to instantiate these types (the concrete implementations of IZptDocument) directly. Instead rely on an implementation of IZptDocumentFactory to create them for you.

ZPT-Sharp supports both XML documents (via the .NET framework's built-in XML support) and HTML documents (via the HTML Agility Pack). These two implementation types deal with these two document-types, and their respective rendering engines.

DefaultRenderingOptions

This is a concrete implementation of RenderingOptions. It is an immutable type (once constructed it may not be altered) - and so all of the available options are constructor parameters. Every parameter is optional, with a default value; a new instance (using a parameterless constructor) provides the default options.
